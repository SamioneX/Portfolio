Project 01

CloudClips

Serverless short-video platform with AI content moderation

Live
↗ Live Demo GitHub →

What it is

A full-stack video sharing platform where users upload short clips (max 5 min), which are automatically transcoded to multiple resolutions, scanned for inappropriate content using machine learning, and served globally via CDN — all without a single always-on server.

Why I built it

"Video platforms deal with the hardest distributed systems problems — async pipelines, large file handling, CDN delivery, content safety at scale. I wanted to build the entire stack, not just the happy path."

Architecture

[ Architecture diagram — draw.io / Excalidraw export goes here ]
S3 → Lambda → SQS → MediaConvert → Rekognition → CloudFront

What I built

  • Presigned S3 upload pattern — browser uploads directly to S3, bypassing the server entirely
  • Event-driven transcoding pipeline via SQS + Lambda + MediaConvert, producing 360p & 720p outputs
  • AWS Rekognition video moderation flagging inappropriate content before publish
  • Amazon Transcribe for auto-generated captions on every video
  • Cognito for user auth; SNS + SES for processing complete notifications

Challenges & learnings

  • Handling async pipeline failures gracefully — implemented a Dead Letter Queue so failed transcoding jobs don't silently vanish
  • Rekognition Video is async; had to design a callback pattern with SNS rather than waiting on a synchronous response
  • CloudFront cache invalidation after video re-processing required careful key design in S3
  • IAM least-privilege setup across 8 services was the most time-consuming but most educational part

Completed Feb 2026 · Solo project · ~5 weeks
